# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# First-time setup
bundle install
npm install

# Development (Jekyll + Tailwind in watch mode)
npm run dev

# Or separately in two terminals:
bundle exec jekyll serve --livereload
npm run css:watch

# Production build
npm run build
```

`assets/css/main.css` is generated by Tailwind and gitignored ‚Äî it must be built before Jekyll can serve the site.

## Architecture

Jekyll 4 static site ¬∑ Tailwind CSS 4 ¬∑ Alpine.js 3 (CDN) ¬∑ German only

### Design tokens
Defined in `assets/css/input.css` via `@theme`:
- `--color-primary: #FF8800` ‚Üí use as `text-primary`, `bg-primary`, `bg-primary/10`, `hover:bg-primary-dark`, etc.
- `--color-primary-dark: #E67A00`

### Content data (`_data/`)
All content that repeats across pages lives here:
- `navigation.yml` ‚Äî main nav links (header + footer)
- `services.yml` ‚Äî 6 service cards with `slug`, `icon`, `short`, `details` (details rendered via `| markdownify`)
- `team.yml` ‚Äî team member profiles
- `welpen.yml` ‚Äî puppy directory (sorted A‚ÄìZ in `welpen.html`)

### Layouts & includes
- `_layouts/default.html` ‚Äî base layout; loads Inter font, Tailwind CSS, Alpine.js CDN, jekyll-seo-tag
- `_layouts/page.html` ‚Äî simple content page with page-hero header section (extends default)
- `_includes/header.html` ‚Äî sticky header with Alpine.js mobile menu (`x-data="{ open: false }"`)
- `_includes/footer.html` ‚Äî dark footer with service list from `_data/services.yml`

### Pages

### Site-wide config (`_config.yml`)
`site.phone` and `site.booking_url` are used in header, footer, and multiple pages. Change them once here.
